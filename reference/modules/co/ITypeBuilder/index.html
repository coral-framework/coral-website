co.ITypeBuilderinterface

Assists in the definition of a new Coral type.

The workflow for defining a new type is as follows:

  1. Create a type builder by calling defineType on a co.INamespace.
  2. Pass data to the type builder via the define methods below, as appropriate for the kind of type you’re defining.
  3. Repeat 1 and 2 as needed to define a graph of types, then use the global transaction to add the graph to the system.

Methods

co.IType createType()

Creates and returns the type. You don’t have to call this method.

This method is automatically called when you commit a type transaction.

  • co.MissingInputException if the type is missing required data.

void defineBaseType(in co.IType baseType)

Makes the type inherit from baseType.

  • co.NotSupportedException if the type doesn’t support inheritance.
  • co.NotSupportedException if the type has already been created.
  • co.NotSupportedException if a base type was already defined.
  • co.IllegalArgumentException if baseType is invalid.

void defineField(in string name, in co.IType type, in bool isReadOnly)

Adds a field to a record type (a struct, interface or native class).

  • co.NotSupportedException if the type is not a record type.
  • co.NotSupportedException if the type has already been created.
  • co.IllegalNameException if name is invalid.
  • co.IllegalArgumentException if type is invalid.
  • co.IllegalArgumentException if isReadOnly is true and kind is TK_STRUCT.

void defineIdentifier(in string name)

Adds an identifier to an enum.

  • co.NotSupportedException if the type is not an enum.
  • co.NotSupportedException if the type has already been created.
  • co.IllegalNameException if name is invalid.

co.IMethodBuilder defineMethod(in string name)

Adds a method to a class type (an interface or native class).

You must use the returned co.IMethodBuilder to complete the definition.

  • co.NotSupportedException if the type is not a class type.
  • co.NotSupportedException if the type has already been created.
  • co.IllegalNameException if name is invalid.

void definePort(in string name, in co.IInterface itf, in bool isFacet)

Adds a port to a component.

  • co.NotSupportedException if the type is not a component.
  • co.NotSupportedException if the type has already been created.
  • co.IllegalNameException if name is invalid.
  • co.IllegalArgumentException if itf is invalid.
