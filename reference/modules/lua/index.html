Lua Modulelua

Integrates the Lua scripting language into Coral.

Untagged

lua.Component

A dynamic component, with interfaces defined in run-time and methods implemented in Lua.

lua.Exception

General-purpose exception raised by the Lua module.

lua.IInterceptor

Observes all interaction between Lua and Coral services.

lua.ILauncher

Main interface for the lua.Launcher component.

lua.IScripted

Annotation for components scripted in Lua.

lua.IState

For interfacing with Lua. The main Lua state is exposed as a global lua.IState service.

lua.Launcher

Launcher component for use with the ‘coral’ launcher executable. This launcher expects a fully-qualified Lua script name as first argument. The script will be invoked with the same argument list, but with the first argument removed.

lua.lua
lua.ModulePart

A co.IModulePart that wraps a Lua script in a module’s dir.

lua.ModulePartLoader

A co.IModulePartLoader that allows Coral modules to be written in Lua. If a module ‘foo’ contains a script named ‘foo.lua’, this component will create a lua.ModulePart to wrap the script as a co.IModulePart.

lua.ScriptedAnnotation

Provides the @lua.Scripted( “script.name” ) annotation.

lua.Universe

Singleton that manages a Lua universe. If multiple instances are created, all will share the same underlying lua_State. The system-wide lua.IState can be obtained as a service.
