co.ITypeTransactioninterface

Atomically validates and adds a set of types to the system.

The transaction validates the integrity of a set of interdependent types as they are added to the system. If the validation fails, it removes all types in the transaction from the system.

Methods

void commit()

Validates and commits all types in the transaction.

This method validates all type builders in the transaction, checking for missing data and semantic errors. If the validation fails, an exception is raised and you must call rollback.

  • co.MissingInputException if a type is missing required data.
  • co.IllegalNameException if a type contains clashing members.
  • co.IllegalStateException if an inheritance cycle is detected.
  • co.NotSupportedException if the transaction is empty or dirty.

void rollback()

Removes references to the types in the transaction.

The types will die with their builders.
