Coral Moduleco

Module that powers the Coral framework.

This module comprises Coral’s three founding pillars:

  1. A service-oriented component model.
  2. A reflective type system.
  3. A module system.

Below is a catalogue of all module types.

System Coral’s bootstrap interface and only singleton

The framework initializes itself the first time you get the co.ISystem singleton.

co.ISystem

Façade to the whole system. Controls the system’s life cycle.

co.SystemState

Enumeration of states in the system’s life cycle.

Component Model Coral’s service-oriented component model

A component in Coral—like a class in OOP—is a template for creating objects. An object is a component instance that provides and consumes services. A service is the basic unit of functionality, and is provided through an interface.

An interface is not only the protocol, but also the contract for a service. Like an object is an instance of a component, we can think of a service as an instance of an interface. Ultimately, both components and interfaces are classes in C++.

Components, besides having a static set of interfaces (at compile time), may provide additional services (at runtime) through dynamic proxies.

co.IDynamicServiceProvider

Allows an object to provide new services at runtime via dynamic proxies.

co.IObject

Provides object identity and reflection for every component instance.

co.IService

Base of all interfaces. Represents a service provided by an object.

co.IServiceManager

A central database where services are published and queried.

Type System Coral’s reflective type system

Coral supports type introspection and reflection throughout the framework, based on the types in this section.

co.INamespace

Represents a namespace in the Coral repository.

co.IReflector

Handles reflection operations for a Coral type.

co.IType

Represents a Coral type. Base interface for type descriptors.

co.ITypeManager

Façade to the type system, with methods to find and create types.

co.TypeKind

Enumeration of Coral type kinds.

Creation Type creation API

co.IMethodBuilder

Assists in the definition of a method for a new Coral type.

co.ITypeBuilder

Assists in the definition of a new Coral type.

co.ITypeTransaction

Atomically validates and adds a set of types to the system.

Annotation Type decoration system

co.IAnnotated

Base interface for services that can be annotated.

co.IAnnotation

Base interface for annotations. Decorates an annotated service.

co.ICppBlock

Contains C++ code for interfaces and native classes in CSL.

co.IDocumentation

Contains documentation for types in CSL.

co.IDynamicTypeProvider

Base interface for annotations that make a type dynamic.

co.IInclude

States a header file must be included in order to use a native class.

co.IncludeAnnotation

Provides the @co.Include annotation for use with native classes.

Introspection Type introspection API

co.IArray

Describes an array type.

co.IClassType

Interface for types that have methods.

co.IComponent

Describes a component type.

co.ICompositeType

Interface for types that have members.

co.IEnum

Describes an enum type.

co.IException

Describes an exception type.

co.IField

Describes a field as a member of a co.IRecordType.

co.IInterface

Describes an interface type.

co.IMember

Interface for members of a co.ICompositeType.

co.IMethod

Describes a method as a member of a co.IClassType.

co.INativeClass

Describes a native class type.

co.IParameter

Describes a method parameter. See co.IMethod.

co.IPort

Describes a component port. See co.IComponent.

co.IRecordType

Interface for types that have fields.

co.IStruct

Describes a struct type.

co.MemberKind

Enumeration of member kinds. See co.IMember.

Module System Coral’s module system

co.IModule

A collection of types and resources.

co.IModuleManager

Façade to the module system, with methods to find and load modules.

co.IModulePart

Represents an artifact loaded as part of a module.

co.IModulePartLoader

Loads a certain kind of co.IModulePart from the repository.

co.ModuleState

Enumeration of states in a module’s life cycle.

Exceptions

co.IllegalArgumentException

Raised when a method is passed an illegal argument.

co.IllegalCastException

Raised when a type cast or a value conversion fails.

co.IllegalNameException

Raised when a name is illegal.

co.IllegalStateException

Raised when something is in an unexpected or illegal state.

co.MissingInputException

Raised when required input is missing.

co.MissingServiceException

Raised when a required service is not available.

co.ModuleLoadException

Raised when an error occurs while loading a module.

co.NoSuchPortException

Raised when a requested port is not available.

co.NotSupportedException

Raised when an operation is not supported.

co.TypeLoadException

Raised when an error occurs while loading a type.

Deprecated

These types are marked as deprecated and will be removed in a future version.

co.CSLError

Contextual information about why a type could not be loaded.

co.Uuid

A 128-bit Universally Unique IDentifier (UUID).
