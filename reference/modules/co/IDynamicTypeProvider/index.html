co.IDynamicTypeProviderinterface

Base interface for annotations that make a type dynamic.

A dynamic type doesn’t require source code generation and is ignored by the Coral compiler. The framework obtains the type’s reflector from this annotation, instead of the type’s module. Thus a dynamic type can be instantiated without even having a module.
