co.ITypeinterface

Represents a Coral type. Base interface for type descriptors.

Types can be identified either by a name within a namespace, or uniquely by fullName. Every type has a kind that determines its actual type and descriptor interface (i.e. subtype of co.IType).

Fields

readonly co.Uuid binarySignature

Fingerprint of the type’s binary interface.

Considers all data that could affect the binary interface of a type—i.e. things that break C++ ABI compatibility.

readonly co.IReflector currentReflector

The type’s reflector, or null if a reflector hasn’t been set yet.

To guarantee you’ll obtain a reflector, get reflector instead.

readonly string fullName

Fully qualified name of the type. For example, “nspc.SomeType”.

readonly co.Uuid fullSignature

Fingerprint of all type data (except annotations).

Considers all data that could affect the source code of a type.

readonly co.TypeKind kind

What kind of type is this type.

readonly string name

Local name of the type (within its namespace). For example, “SomeType”.

readonly co.INamespace namespace

Namespace where the type is defined.

co.IReflector reflector

The type’s reflector. Getting this field may raise an exception.

If the reflector is null, the getter will attempt to load the type’s module in order to get a reflector. If it’s still null then, the getter will raise an exception. The setter is plain and safe, however.
