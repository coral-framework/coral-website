co.IModulePartinterface

Represents an artifact loaded as part of a module.

A module part represents an artifact—such as a library, script or data file—that adds functionality to the system.

This interface plays a central role in program composition, since it ultimately handles all system life-cycle events.

See co.IModule for an explanation of all module events.
