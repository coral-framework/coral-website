co.IModuleinterface

A collection of types and resources.

A module consists in a set of parts loaded from a namespace. Each module part represents an artifact—such as a library, script or data file—that adds functionality to the system.

Modules and their parts share the same life cycle, which is intertwined with the system’s life cycle. Whenever the system changes state it updates all modules, and whenever a module changes state it updates its parts.

Modules must respond to five life-cycle events:

  1. In initialize modules should publish services and set reflectors for all their types. During this event a module cannot do anything that requires other modules—it should wait until integrate.
  2. In integrate modules may query and use services from other modules.
  3. In integratePresentation modules can integrate presentation elements.
  4. In disintegrate modules should release all external references.
  5. In dispose modules should free all allocated resources.

The system will abort a module if any event raises an exception.

Fields

readonly co.INamespace namespace

Namespace where the module is located.

readonly co.IModulePart[] parts

List of parts that comprise the module.

int32 rank

Adjusts the order in which modules receive updates from the system.

Modules are sorted by increasing rank value. The default rank is zero.

readonly co.ModuleState state

Current state of the module and all its parts.

Methods

void abort()

Aborts the module.

If the module was already integrated, this method calls disintegrate.

This always calls dispose and changes state to ModuleState_Aborted.

  • co.IllegalStateException if the module was already disposed.

void disintegrate()

Disintegrates the module from the system.

This changes state to ModuleState_Disintegrated.

  • co.IllegalStateException if state isn’t ModuleState_PresentationIntegrated.

void dispose()

Disposes the module.

This changes state to ModuleState_Disposed.

  • co.IllegalStateException if state isn’t ModuleState_Disintegrated.

void initialize()

Initializes the module.

This changes state to ModuleState_Initialized.

  • co.IllegalStateException if state isn’t ModuleState_None.

void integrate()

Integrates the module with the system.

This changes state to ModuleState_Integrated.

  • co.IllegalStateException if state isn’t ModuleState_Initialized.

void integratePresentation()

Integrates the module’s presentation into the system.

This changes state to ModuleState_PresentationIntegrated.

  • co.IllegalStateException if state isn’t ModuleState_Integrated.
